# 天气数据采集 · 官方文档规则

## 数据源：Open-Meteo 免费API

### 官方规则（来自 open-meteo.com/en/terms）

| 限制 | 数值 | 说明 |
|------|------|------|
| 日限额 | **10,000次/天** | 非商业用途免费 |
| 时限额 | 5,000次/小时 | — |
| 分钟限额 | 600次/分钟 | — |
| API Key | **不需要** | 非商业用途 |
| 归因 | CC-BY 4.0 | 标注"数据来自Open-Meteo" |

### 历史数据API

| 项目 | 内容 |
|------|------|
| 端点 | `https://archive-api.open-meteo.com/v1/archive` |
| 模型 | **ERA5** (0.25°, 1940至今) — 官方推荐长期气候研究用这个 |
| 分辨率 | 逐小时或逐日 |
| 一次请求 | 可拉**多年数据**（单次请求，不限时长） |

### 参数规范

| 参数 | 值 | 说明 |
|------|-----|------|
| latitude | 产区纬度 | 每个产区一个请求 |
| longitude | 产区经度 | — |
| start_date | 2011-01-01 | 与价格数据对齐 |
| end_date | 今天 | 后续增量更新 |
| daily | temperature_2m_mean,precipitation_sum | 日温度+降水即可 |
| timezone | Asia/Shanghai | 中国时区 |

## 采集策略（严格遵守官方规则）

### 为什么逐步拉？

| 原因 | 说明 |
|------|------|
| 单次请求可拉多年 | ERA5归档API一次调取不限天数 |
| 6个产区 = 6次请求 | 远远低于10,000次/天限额 |
| 每次请求带不同产区 | 每个产区一个独立请求，错误隔离 |

### 六大核心产区

| 产区 | 纬度 | 经度 | 品种 | 关键期 |
|------|------|------|------|--------|
| 美国中西部 IL | 40.0 | -89.0 | 大豆/玉米 | 6-8月 |
| 美国中西部 IA | 42.0 | -93.0 | 大豆/玉米 | 6-8月 |
| 巴西 MT | -13.0 | -55.0 | 大豆 | 12-2月 |
| 巴西 PR | -24.0 | -51.0 | 大豆 | 12-2月 |
| 阿根廷 | -33.0 | -62.0 | 大豆 | 12-2月 |
| 美国 KS | 38.5 | -98.0 | 小麦 | 3-5月 |

### 采集流程

```
第1步：拉美国IL（2011-2026）→ 写入 agri_weather
第2步：拉美国IA（2011-2026）→ 追加
第3步：拉巴西MT（2011-2026）→ 追加
第4步：拉巴西PR（2011-2026）→ 追加
第5步：拉阿根廷（2011-2026）→ 追加
第6步：拉美国KS（2011-2026）→ 追加
第7步：每日增量更新（只拉昨天到今天）
```

### 错误处理

| 情况 | 处理 |
|------|------|
| 429限流 | 等60秒重试，不连续轰炸 |
| 某产区失败 | 跳过，记录日志，下一个 |
| 数据为空 | 标注缺失，不编造 |
