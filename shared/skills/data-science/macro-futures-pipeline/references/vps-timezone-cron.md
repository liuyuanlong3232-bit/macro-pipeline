# VPS时区和cron时间配置

## 时区
- VPS原时区：Etc/UTC (UTC+0)
- 修改后：Asia/Shanghai (UTC+8)
- 修改命令：`timedatectl set-timezone Asia/Shanghai`
- 原因：用户在中国，cron任务需要按中国时间执行

## 数据源发布时间（北京时间）
| 数据源 | 发布时间 | 说明 |
|--------|---------|------|
| FRED | 每日（不定时） | 美联储经济数据 |
| Yahoo期货 | 交易日开盘期间 | 黄金/白银/原油 |
| EIA能源 | **周三 22:30** | 原油库存/产量 |
| COT持仓 | **周五 03:30** | CFTC持仓报告 |
| USDA农业 | 周五（不定时） | 农产品数据 |

## 数据采集cron（中国时间）
| 时间 | 任务 | 数据源 |
|------|------|--------|
| 每日 08:00 | FRED/Yahoo采集 | 当日最新数据 |
| 周三 23:00 | EIA采集（发布后30分钟） | 周三EIA数据 |
| 周五 04:00 | COT采集（发布后30分钟） | 周五COT数据 |

## 报告cron（中国时间）
| 时间 | 报告 | 备注 |
|------|------|------|
| 周一 09:00 | 宏观周报 | FRED数据已采集 |
| 周四 09:00 | 能源周报 | EIA数据已采集（周三23:00） |
| 周五 09:00 | 国际农业 | USDA数据已采集 |
| 周五 20:00 | 中国农业 | Tushare收盘数据 |
| 周六 09:00 | 贵金属 | COT数据已采集（周五04:00） |
| 周日 10:00 | 资产配置 | 综合所有数据 |

## 关键经验
- VPS默认UTC时间，中国用户必须改为Asia/Shanghai
- cron任务时间基于数据源发布时间，不是随意设定的
- EIA/COT数据要在发布时间后立即采集，确保报告最新
- 数据非实时（滞后1-2天），monitor.py已移除，无意义
