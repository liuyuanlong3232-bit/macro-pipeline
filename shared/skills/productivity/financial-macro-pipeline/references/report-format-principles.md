# Report Format Principles

These rules MUST be followed for all financial/commodity market reports. They were explicitly specified by the user and are non-negotiable.

## Six Hard Rules

1. **Data First** — Lead with numbers, not narrative. Every section opens with the relevant metric.
2. **No Price Targets** — Never predict specific levels (e.g., "gold will reach $X"). Do not include implied targets even indirectly via ranges.
3. **No Trading Advice** — Never use: buy, sell, long, short, accumulate, reduce, add to position, enter, exit, take profit.
4. **No Action Plans** — Never recommend: "should consider", "ideal entry", "best to wait", "monitor for", "look to".
5. **Every Data Point Has a Date** — Format: `Metric: Value (YYYY-MM-DD)`. No date = omission.
6. **Every Data Point Has a Source** — Format-like: `Source: FRED / CFTC / Alpha Vantage / Finnhub`.

## Permitted vs Forbidden

| Permitted | Forbidden |
|-----------|-----------|
| "Gold rose/feel by X%" | "Gold is likely to rise further" |
| "Positioning shows X" | "This supports a bullish view" |
| "TIPS at Y%, the real rate environment" | "Falling real rates are bullish for gold" |
| "CFTC spec long/short ratio" | "Specs are building a bullish case" |
| "Geopolitical event Z occurred" | "Geopolitical risk may push gold higher" |
| "This week economic calendar includes" | "Key events to watch for direction" |

## Weekly Report Scope
Weekly reports may include additional sections:
- **Market Pricing**: How much of each risk factor is priced in
- **Cycle Position**: 1w / 1m / 3m / 6m / 1y view
- **Scoring System**: Macro, real rates, dollar, ETF flows, CFTC, central bank buying, supply-demand
- **Next Week Calendar**: Scheduled data releases and events

Even in weekly reports, rules 1-6 still apply. The scoring system is descriptive, not predictive — it summarizes current conditions, not forecasts.
