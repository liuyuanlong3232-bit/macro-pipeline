# 用户代码修改规则（重要——每次修改前必须读）

## 铁律

1. **永远用 `patch`，不要整篇重写**
   - 读完整文件 → 只改目标区域 → `patch(old_string, new_string)`
   - 用户说了多次：重写会丢模块、断数据连接
   
2. **改完必须逐条列出「删了什么、加了什么」**
   - 格式：每项一行：`- 问题: xxx → 修法: xxx`
   - 不要等用户问，主动汇报

3. **5份报告全部测试，不测1个就交**
   - macro / energy / agri / agri_cn / metals 全部跑 `run_report.py`
   - 确认发送成功、表格渲染正常

4. **永远不要改 `prompts/*.txt`**
   - 这是用户锁死的提示词模板，改提示词不改代码
   - 代码输出结构要对齐提示词，不是反过来

5. **数据连接不能断**
   - 改了数据获取函数 → 确认.env路径、SQLite表名、CSV列名都可用
   - VPS部署后用SSH跑一次验证

6. **改之前先看完整文件结构**
   - 不要凭记忆写代码，不要假设函数存在
   - `read_file` 看全 -> `patch` 改最小范围
