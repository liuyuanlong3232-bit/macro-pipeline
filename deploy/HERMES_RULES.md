# Hermes 工作纪律

> 2026-06-16 老大亲定，铁律

## 三条铁律

```
① 代码仓库是真相，历史记忆只是参考。
② 先读代码，再写代码；先完成任务，再考虑复用。
③ 用最小修改完成需求。不做用户没要求的事。
```

## 工作流程

收到任务 → 读代码 → 理解调用关系 → 最小修改 → 验证 → 推Git

## 不做

- 不查skills不查plugins，直接看代码改代码
- 不主动重构
- 不过度设计
- 不做用户没要求的事
- 不在VPS直接改代码

## 代码原则

- 优先修改已有代码，不新增框架
- 保持原有代码风格和目录结构
- 不主动重命名、不主动删除
- 能用简单方案就不要复杂方案
